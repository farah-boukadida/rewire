# 场景：一人扛海外运营，跨时区 24 小时响应

> **用户描述**：我远程工作，一个人负责海外市场运营。每天处理不同时区客户的邮件和消息（Gmail + Slack），监控社媒账号数据（Twitter/Instagram），整理日报发给国内团队。因为时差问题，客户半夜发的消息我第二天才能看到，经常错过最佳响应时间。希望有 Agent 能 24 小时帮我盯着，我只在需要判断的时候介入。

---

## 你现在的问题

> 你不是在做运营，你是在当一个跨时区的人肉路由器——凌晨 3 点欧洲客户发的邮件要等你早上 9 点起来才能转，而他已经在等了 6 小时。你的瓶颈不是能力，是你的生物钟。一个人的时区覆盖不了 24 小时的地球，但一台不睡觉的电脑可以。

---

## 闭环拆解

你的日常工作其实是三条并行的闭环在跑，但全部卡在你一个人身上：

### 闭环 A：客户消息响应（Gmail + Slack）

| 步骤 | 现在谁在做 | 具体动作 | 状态 |
|------|-----------|---------|------|
| **调研** | 人肉 | 早上起来逐个打开 Gmail、Slack，翻阅过夜消息 | 意愿瓶颈（最怕打开收件箱） |
| **决策** | 人肉 | 判断每条消息的优先级、需不需要回复、怎么回 | 核心判断，但 70% 是常见问题 |
| **规划** | 人肉 | 想好回复内容、决定先回哪个 | 与决策合并 |
| **执行** | 人肉 | 逐条写回复，发出去 | 吞吐量瓶颈（量大时 1-2 小时） |
| **反馈** | 被动 | 对方回了才知道有没有解决问题 | 延迟 6-12 小时 |
| **复盘** | 缺失 | 哪些是高频问题？响应时间是否在改善？ | 完全没有数据 |

### 闭环 B：社媒监控（Twitter / Instagram）

| 步骤 | 现在谁在做 | 具体动作 | 状态 |
|------|-----------|---------|------|
| **调研** | 人肉 | 打开 Twitter/Instagram 后台，看数据面板 | 纯登录搬运 |
| **决策** | 人肉 | 哪个帖子数据好？要不要追加推广？ | 有判断价值 |
| **规划** | 缺失 | 没有系统化的监控基线 | 凭感觉 |
| **执行** | 人肉 | 手动记录数据到表格 | 意愿瓶颈（最无聊的活） |
| **反馈** | 缺失 | 数据变化没有告警 | 爆帖子了也不知道 |
| **复盘** | 缺失 | 没有趋势分析 | 数据记了也没看 |

### 闭环 C：日报整理

| 步骤 | 现在谁在做 | 具体动作 | 状态 |
|------|-----------|---------|------|
| **调研** | 人肉 | 回忆今天干了啥，翻聊天记录、邮件 | 耗时且遗漏 |
| **决策** | 人肉 | 哪些值得汇报 | 判断价值低 |
| **规划** | 人肉 | 想日报结构 | 每天重复 |
| **执行** | 人肉 | 写日报、发到国内群 | 吞吐量瓶颈 |
| **反馈** | 被动 | 国内团队偶尔问一句 | 单向 |
| **复盘** | 缺失 | 日报质量从未被评估 | 写了就完了 |

三条闭环共 18 步，15 步人肉，3 步缺失。你一个人在做 18 个人的活，而且时区让你物理上不可能 24 小时在线。

## Claude Code 怎么驱动这一切

**机制**：Claude Code 运行在你的电脑（或一台常开的 Mac mini / 云服务器）上，通过 Google Workspace CLI 操作 Gmail，通过 agent-slack CLI 读写 Slack 消息，用 bb-browser 操作 Twitter/Instagram 后台页面。全部是 CLI 工具，Claude Code 直接在 shell 里调用。它不是一个需要你坐在电脑前才能动的工具——配合 `launchd`（macOS）或 `cron`（Linux），它可以每隔 15 分钟自动醒来检查一次，24 小时不间断。你睡觉时，它替你值夜班。

**故事**：

凌晨 2 点，一个美国客户在 Slack 里问："我们的 API 集成文档在哪？"

Claude Code 15 分钟后自动醒来，通过 agent-slack CLI 读到这条消息。它先查你在 CLAUDE.md 里定好的规则——"API 文档类问题"属于 FAQ，对应的标准回复是文档链接加一句引导语。它直接调 Slack API 回了一条：

> "Hi! Here's our API integration guide: [链接]. If you need help with a specific endpoint, feel free to ask and I'll flag it for our team."

同时它在你的 Bark 上推了一条通知："已自动回复 @john 的 API 文档咨询"，声音设成最轻，不吵醒你。

凌晨 4 点，一个欧洲客户发了一封 Gmail 邮件，语气不太好："We've been waiting for three days for an update on our billing issue."

Claude Code 通过 Google Workspace CLI 读到这封邮件，判断这是「客户投诉 + 计费问题」——规则里写着：涉及钱的事不能自动回复，标记为高优先级。它做了两件事：给你发一条 Bark 推送，level 设为 `timeSensitive`："紧急：欧洲客户投诉计费问题，等待 3 天未回复，需要你处理"；同时自动起草了一封安抚邮件存为 Gmail 草稿：

> "Thank you for your patience. I'm looking into this with our billing team right now and will update you within 24 hours."

你早上 8 点起来，手机上已经有分好类的通知摘要。那条计费投诉排在最上面，你打开 Gmail，草稿已经写好了，扫一眼内容没问题，点发送，30 秒搞定。其他 6 条消息已经被自动回复了，你只需要看一眼回复记录确认没问题。

与此同时，Claude Code 在凌晨 5 点还跑了一轮社媒巡检。它用 bb-browser 打开你的 Twitter Analytics 页面，截图保存，提取关键数据——昨天发的那条产品更新推文获得了 2.3 万展示量，比平均值高 4 倍。它在 Instagram 也发现一条 Reel 的完播率突破了 35%。这些数据被自动整理到一个本地 JSON 文件里。

到了你设定的日报时间（比如北京时间下午 3 点），Claude Code 自动启动，汇总当天所有数据：处理了多少条消息、自动回复了几条、有几条需要人工跟进、社媒各平台的核心指标变化、异常数据高亮。然后它把这份日报格式化成 Markdown，通过飞书 Webhook 或 Slack 发到国内团队的群里。

你的整个工作流程从"起床→打开 10 个标签页→花 3 小时逐条处理→再花 1 小时写日报"，变成了"起来看一眼手机通知→处理 2-3 条需要判断的事→日报已经自动发了"。

## 你的工作会变成什么样

早上 8 点，你还没起床，手机 Bark 通知已经整整齐齐地列着：

- "过夜消息摘要：共 8 条。6 条已自动回复，2 条需人工处理（1 条计费投诉、1 条合作咨询）"
- "社媒日报：Twitter 推文展示量 +340%（异常高，建议追加互动），Instagram Reel 完播率 35%"

你打开手机，看投诉邮件的草稿，改了一个措辞，发出去。合作咨询的邮件需要想一想，你先标记"待处理"，洗漱完再说。

9 点坐到电脑前，跟 Claude Code 说："合作咨询那封怎么回？帮我看看对方公司背景。"它用 bb-browser 搜了对方公司，2 分钟给你一页摘要，你拍板了回复方向，它起草邮件，你确认发出。

下午 3 点，日报自动发到国内群里了，格式比你手写的还清楚。老板回了个"收到"。

你今天真正用脑子的时间：30 分钟。剩下的时间，你可以去想"下一个月的海外推广策略怎么调整"——这才是运营的活。

## 怎么搭建

### 1. 准备工具

```bash
# bb-browser（浏览器自动化，操作 Twitter/Instagram 后台）
bb-browser status  # 确认 daemon 在跑

# Google Workspace CLI（Gmail / Drive / Calendar 等，Google 官方出品，12.4k stars）
# 仓库：https://github.com/googleworkspace/cli
npm install -g @anthropic/google-workspace-cli  # 按仓库文档安装配置 OAuth

# agent-slack（专为 AI Agent 设计的 Slack CLI，26 分钟前仍在活跃开发）
# 仓库：https://github.com/stablyai/agent-slack
npm install -g agent-slack
# 配置 Slack Bot Token（需要 channels:read, channels:history, chat:write 等权限）

# 飞书 Webhook（发日报到国内团队群）
# 在飞书群设置里添加"自定义机器人"，拿到 Webhook URL

# jq（JSON 处理）
brew install jq
```

### 2. 创建项目目录

```bash
mkdir ~/overseas-ops-agent && cd ~/overseas-ops-agent
mkdir -p data/social logs
```

### 3. 编写 CLAUDE.md

```markdown
# 海外运营 24 小时 Agent

## 身份
你是海外市场运营助手，负责 24 小时监控客户消息和社媒数据。

## 工具
- Google Workspace CLI（https://github.com/googleworkspace/cli）：读写 Gmail（读邮件、起草回复、发送）
- agent-slack CLI（https://github.com/stablyai/agent-slack）：读写 Slack 消息（读未读消息、发送回复）
- bb-browser：操作 Twitter/Instagram 后台（截图、提取数据）
- curl：调飞书 Webhook 发日报
- Bark：给我发手机通知

## 客户消息处理规则

### 自动回复（无需人工确认）
- FAQ 类问题：按下方 FAQ 库直接回复
  - API 文档 → [文档链接] + 引导语
  - 定价咨询 → [定价页链接] + 约 demo
  - 账号问题 → [帮助中心链接] + 提交工单引导
  - 功能咨询 → [对应功能文档] + 使用建议
- 简单确认类："收到"、"Thanks"、"Got it" → 不回复，标记已读

### 起草但不发送（存草稿 + 通知我）
- 客户投诉 / 情绪负面
- 涉及金钱：退款、计费、合同
- 合作/商务咨询
- 技术问题无法确认答案
- 任何拿不准的

### 通知规则
- 自动回复后 → Bark 推送，level: passive
- 起草待审 → Bark 推送，level: active
- 紧急投诉/大客户 → Bark 推送，level: timeSensitive
- 系统异常 → Bark 推送，level: critical

## 社媒监控

### Twitter
- 打开 https://analytics.twitter.com（或 x.com/analytics）
- 截图保存到 data/social/twitter-{date}.png
- 提取：推文展示量、互动量、粉丝变化
- 异常判断：任何指标偏离 7 日均值 ±50% 标记为异常

### Instagram
- 打开 Instagram Insights 页面（bb-browser 操作）
- 截图保存到 data/social/ig-{date}.png
- 提取：Reach、互动率、Reel 完播率、粉丝变化
- 异常判断：同上
```

### 7. 配置 24 小时定时运行

验证交互模式没问题后，配置 `launchd` 让它每 15 分钟自动跑一次：

```bash
# 创建巡检脚本
cat > ~/overseas-ops-agent/patrol.sh << 'EOF'
#!/bin/bash
cd ~/overseas-ops-agent
claude --print --dangerously-skip-permissions \
  -p "执行定时巡检：1) 检查 Gmail 和 Slack 新消息并按规则处理 2) 记录处理结果到 logs/patrol-$(date +%Y%m%d-%H%M).log" \
  2>&1 | tee -a logs/patrol-$(date +%Y%m%d-%H%M).log
EOF
chmod +x ~/overseas-ops-agent/patrol.sh

# 创建社媒巡检脚本（每天跑两次就够）
cat > ~/overseas-ops-agent/social-check.sh << 'EOF'
#!/bin/bash
cd ~/overseas-ops-agent
claude --print --dangerously-skip-permissions \
  -p "执行社媒数据采集：用 bb-browser 打开 Twitter 和 Instagram 后台，截图并提取关键数据，保存到 data/social/" \
  2>&1 | tee -a logs/social-$(date +%Y%m%d-%H%M).log
EOF
chmod +x ~/overseas-ops-agent/social-check.sh

# 创建日报脚本
cat > ~/overseas-ops-agent/daily-report.sh << 'EOF'
#!/bin/bash
cd ~/overseas-ops-agent
claude --print --dangerously-skip-permissions \
  -p "生成今日运营日报：汇总 logs/ 里今天的巡检记录和 data/social/ 里的社媒数据，按日报模板整理，通过飞书 Webhook 发送" \
  2>&1 | tee -a logs/report-$(date +%Y%m%d).log
EOF
chmod +x ~/overseas-ops-agent/daily-report.sh
```

macOS `launchd` 定时任务配置：

```xml
<!-- ~/Library/LaunchAgents/com.ops.patrol.plist -->
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key>
    <string>com.ops.patrol</string>
    <key>ProgramArguments</key>
    <array>
        <string>/bin/bash</string>
        <string>/Users/你的用户名/overseas-ops-agent/patrol.sh</string>
    </array>
    <key>StartInterval</key>
    <integer>900</integer> <!-- 每 15 分钟 -->
    <key>StandardOutPath</key>
    <string>/Users/你的用户名/overseas-ops-agent/logs/launchd-out.log</string>
    <key>StandardErrorPath</key>
    <string>/Users/你的用户名/overseas-ops-agent/logs/launchd-err.log</string>
    <key>RunAtLoad</key>
    <true/>
</dict>
</plist>
```

```bash
# 加载定时任务
launchctl load ~/Library/LaunchAgents/com.ops.patrol.plist

# 社媒巡检用 crontab，每天 6:00 和 18:00 各跑一次
crontab -e
# 添加：
# 0 6,18 * * * /Users/你的用户名/overseas-ops-agent/social-check.sh
# 0 7 * * * /Users/你的用户名/overseas-ops-agent/daily-report.sh  # 北京时间 15:00（假设你在 UTC+0）
```

## 运行模式：三档并行

这个场景天然适合**多档混合运行**：

| 模式 | 触发方式 | 覆盖场景 | 你的介入 |
|------|---------|---------|---------|
| **24 小时无人值守** | launchd 每 15 分钟 | 消息巡检 + FAQ 自动回复 | 看 Bark 通知即可 |
| **定时模式** | cron 每天 2 次 | 社媒数据采集 + 日报生成 | 不介入 |
| **交互模式** | 你主动启动 | 草稿审核、复杂回复、策略讨论 | 你坐在电脑前对话 |

关键：这三档不冲突。无人值守处理 80% 的日常，定时任务处理数据采集，你只在需要判断力的时候坐下来。

## 元层：它会越来越好

**FAQ 库扩充**：每次你手动回复一条消息，把问题和你的回复告诉 Claude Code："这个以后自动回复。"它帮你把新条目加到 CLAUDE.md 的 FAQ 库里。第一周可能只有 10 条 FAQ，一个月后变成 50 条，自动回复率从 30% 涨到 80%。

**分类规则校准**：如果你发现某条消息被错误地自动回复了（比如把投诉当成了咨询），告诉它。它更新判断规则，同类错误不再发生。每周看一次自动回复记录，花 10 分钟校准，比你每天花 2 小时手动回复划算得多。

**响应时间追踪**：Agent 自动记录每条消息的到达时间和回复时间。每周生成一份响应时间报告——改造前平均 6 小时，改造后 FAQ 类 15 分钟内、人工类 2 小时内。这个数据本身就是你向老板证明价值的武器。

**社媒基线学习**：Agent 积累一个月的数据后，自动建立每个指标的 7 日滚动基线。异常检测从"偏离 50%"逐步调整到更贴合你账号实际波动的阈值。它还会开始发现规律："每周三发的内容互动率普遍高 20%"——这些 insight 以前你根本没时间去算。

**日报迭代**：国内团队如果反馈"能不能加上竞品动态"或"按地区分消息来源"，你改一行 CLAUDE.md 就生效。日报格式从第一版的粗糙逐步演化成团队真正想看的样子。
