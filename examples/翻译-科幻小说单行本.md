# 场景：全自动英译中科幻小说——一个脚本翻完一本书

> **用户描述**：我想自动化翻译英文科幻小说单行本。不是我翻它校对——是让 Claude Code 自己翻译整本书。一本 300 页的书远超单次上下文窗口，需要拆成多个 subagent 翻译，但跨章术语、人名、风格必须一致。流水线：拆书→翻译→校对→润色，每个环节可以用不同的 subagent。术语表在翻译过程中自动积累，前面章节确定的译法传递给后续章节。最终输出排版好的中文 EPUB/Markdown。这是一个工程问题。

---

## 你现在的问题

> 翻译一本书的难点不是翻某一段——Claude 单次就能翻得不错。难点是**300 页塞不进一个上下文窗口**，一旦拆成独立的章节分别翻，每一章都变成了失忆状态：第三章的 subagent 不知道第一章把 "the Drift" 翻成了"漂流区"，第八章突然把主角名字换了写法。人类译者靠脑子记、靠回翻前文保持一致，但你要让机器做到这一点，就需要一套**显性的跨章记忆系统**和一条**可重跑的流水线**。这是一个编排问题，不是一个提示词问题。

---

## 闭环拆解

翻译一本书的完整闭环：

| 步骤 | 人类译者怎么做 | 自动化的挑战 | 方案 |
|------|--------------|-------------|------|
| **调研** | 拿到 EPUB/PDF，通读全书，理解世界观 | 全书太长，无法一次读完 | 先提取目录和前三章做世界观摘要，生成初始术语表 |
| **决策** | 确定翻译风格、术语译法 | 需要在翻译开始前做出关键决策 | 分析阶段生成风格指南和术语种子表，人确认后锁定 |
| **规划** | 脑子里排章节顺序 | 需要编排 subagent 的依赖关系 | 主脚本控制流水线，严格按章节顺序串行翻译 |
| **执行** | 逐章翻译 | 单章可行，跨章一致性是难题 | 每章一个 subagent，携带术语表 + 前文摘要 + 风格指南 |
| **反馈** | 校对——逐句对比中英文 | 同样需要理解上下文 | 独立的校对 subagent，输入原文 + 译文 + 术语表 |
| **复盘** | 通读全书润色 | 全书仍然塞不进上下文 | 逐章润色 + 全书一致性扫描 |

核心工程挑战：**上下文窗口是硬限制，但跨章一致性是硬需求。** 解法是把"记忆"从上下文里抽出来，变成文件系统上的显性状态——术语表、风格指南、前文摘要——让每个 subagent 都能读到。

---

## Claude Code 怎么驱动这一切

**机制**：写一个 shell 脚本，用 `claude` CLI 的 `--print -p` 模式批量调度 subagent。每个 subagent 是一次独立的 `claude` 调用，读取项目目录里的指令文件和状态文件，完成一个具体任务（翻译一章、校对一章、润色一章），然后把结果写回文件系统。subagent 之间不共享上下文窗口，但**共享文件系统**——术语表、风格指南、章节摘要都是磁盘上的文件，这就是跨章记忆的载体。

整条流水线分五个阶段，全部由一个主脚本 `translate.sh` 驱动：

**故事**：

你拿到一本 320 页的科幻小说 EPUB，放进 `source/` 目录。终端里敲一行命令：

```bash
./translate.sh source/book.epub
```

脚本开始跑。

**第一阶段：拆书。** 脚本调用 `pandoc` 把 EPUB 转成 Markdown，然后用一个 subagent 按章节标题拆分成 `chapters/ch01-en.md` 到 `chapters/ch24-en.md`，共 24 章。这步 30 秒搞定。

**第二阶段：分析。** 一个"分析师" subagent 读取全书前三章和目录，产出三样东西：
- `glossary.jsonl`——术语种子表，包含前三章出现的所有专有名词（人名、地名、科技术语、虚构概念）及建议译法
- `style-guide.md`——风格指南，包括叙事人称、时态、语体风格（硬科幻偏书面 vs 太空歌剧偏口语）、特殊处理规则
- `world-summary.md`——世界观摘要，核心设定、势力关系、时间线

这三个文件就是整本书的"翻译宪法"，后续每个 subagent 都会读取它们。如果你想在这里介入——比如觉得某个人名的译法不好——可以直接改文件，再继续跑。

**第三阶段：翻译。** 这是最重的环节。脚本**按章节顺序串行**启动翻译 subagent（不能并行，因为每章会产出新术语，后续章节要用）。每个翻译 subagent 收到四样东西：

1. 本章原文（`chapters/ch05-en.md`）
2. 当前术语表（`glossary.jsonl`）——包含前四章积累的所有术语
3. 风格指南（`style-guide.md`）
4. 上一章的摘要（`summaries/ch04-summary.md`）——200 字的情节摘要，用于衔接上下文

subagent 翻完一章后，输出三样东西：
1. 本章译文（`chapters/ch05-zh.md`）
2. 本章新术语（追加到 `glossary.jsonl`）
3. 本章摘要（`summaries/ch05-summary.md`）——供下一章使用

第 5 章翻完，第 6 章的 subagent 启动时，术语表里已经有了前五章积累的全部术语。这就是"滚雪球式"的术语传递——不需要上下文窗口记住前文，文件系统就是记忆。

24 章翻完，大约需要 40-60 分钟（取决于章节长度和 API 响应速度）。

**第四阶段：校对。** 这一轮可以**并行**——因为术语表已经定稿了。脚本同时启动多个校对 subagent（比如 4 个并行），每个 subagent 拿到一章的原文、译文和术语表，做三件事：
- 漏译检查：原文有但译文没有的句子
- 术语一致性：译文中的术语是否与术语表一致
- 语义准确性：有没有翻反、翻错、误解上下文的地方

每章输出一份校对报告（`reviews/ch05-review.md`），如果有需要修改的地方，直接输出修改后的译文（`chapters/ch05-zh.md` 覆盖）。

4 路并行，24 章大约 15-20 分钟。

**第五阶段：润色。** 同样可以并行。润色 subagent 只关注中文的自然度：消灭翻译腔、拆解欧化长句、让对话读起来像人话。它读取风格指南，确保润色后的风格统一。输出最终译文到 `output/ch05-zh.md`。

最后，脚本把所有章节合并成一个完整的 Markdown 文件（`output/book-zh.md`），再用 `pandoc` 转成 EPUB（`output/book-zh.epub`）。

全程你可以不在电脑前。跑完后 Bark 推一条通知："《The Drift》翻译完成，24 章，67,000 字，EPUB 已生成。"

### 文件系统即记忆

```
│   ├── analyze.md             # 分析阶段 prompt

│   ├── proofread.md           # 校对阶段 prompt

│   └── ...
├── state/                     # 跨章状态（记忆的载体）
│   ├── glossary.jsonl         # 术语表（滚雪球积累）
│   ├── style-guide.md         # 风格指南（分析阶段产出）
│   ├── world-summary.md       # 世界观摘要
│   └── summaries/
│       ├── ch01-summary.md    # 各章情节摘要（翻译阶段产出）
│       └── ...
├── reviews/                   # 校对报告
│   ├── ch01-review.md
│   └── ...
└── output/                    # 最终产出
    ├── ch01-zh.md             # 润色后的各章
    ├── book-zh.md             # 合并的完整译文
    └── book-zh.epub           # 最终 EPUB
```

### 为什么术语表用 JSONL

```jsonl
{"en":"quantum entanglement drive","zh":"量子纠缠驱动器","category":"tech","note":"核心推进系统，全书统一","first":"ch01"}
{"en":"Kepler Station","zh":"开普勒空间站","category":"location","note":"故事主要场景","first":"ch01"}
{"en":"Elena Vasquez","zh":"埃琳娜·巴斯克斯","category":"character","note":"女主角，舰长","first":"ch01"}
{"en":"the Drift","zh":"漂流区","category":"location","note":"危险星域，不译为'漂移'","first":"ch02"}
```

- 每行一个条目，追加写入不需要解析整个文件
- subagent 可以用 `cat` 直接读取全部内容，体积小（通常 < 500 行）
- `category` 字段方便按类型过滤（`character` / `location` / `tech` / `concept`）
- `first` 字段记录首次出现的章节，方便溯源

### 为什么翻译必须串行，校对可以并行

翻译阶段有**数据依赖**：第 N 章的翻译会产出新术语，第 N+1 章必须使用更新后的术语表。如果并行翻译，两个 subagent 可能对同一个术语给出不同译法，术语表产生冲突。

校对和润色阶段**没有数据依赖**：术语表在翻译阶段已经定稿，每章的校对/润色是独立的，可以 4-8 路并行，大幅缩短总时间。

### 流水线时间估算

| 阶段 | 并行度 | 单章耗时 | 24 章总耗时 |
|------|--------|---------|------------|
| 拆书 | - | 30 秒 | 30 秒 |
| 分析 | 1 | 3-5 分钟 | 3-5 分钟 |
| 翻译 | 1（串行） | 2-3 分钟/章 | 48-72 分钟 |
| 校对 | 4 并行 | 2 分钟/章 | 12-15 分钟 |
| 润色 | 4 并行 | 1-2 分钟/章 | 6-12 分钟 |
| 合并+EPUB | - | 1 分钟 | 1 分钟 |
| **总计** | | | **~70-105 分钟** |

一本 300 页的书，不到两小时全自动翻完。

## 你的工作会变成什么样

你拿到一本想翻的英文科幻小说 EPUB。把文件丢进目录，跑一行命令，去吃个饭。回来的时候手机上有一条通知："翻译完成"。你打开 `output/book-zh.epub`，在阅读器里从头翻一遍。

因为有术语表锁定，全书 24 章里"开普勒空间站"不会突然变成"开普勒站"，"漂流区"不会突然变成"漂移带"。因为有风格指南，硬科幻的技术描写段保持了冷峻感，不会突然跳到轻小说腔。因为有前文摘要传递，跨章的人物关系、情节因果没有断裂。

你通读一遍，标记几处不满意的——也许某个双关语的处理太直白了，也许某段对话的语气不够紧张。你改完这些，就是终稿。

从"一本书 2-3 个月"变成"跑 2 小时 + 你通读润色半天"。你的角色从"译者"变成了"审校"——审的是成品，不是从零开始翻。

## 怎么搭建

### 1. 准备工具

```bash
# pandoc（文档格式转换，EPUB ↔ Markdown）
brew install pandoc

# Calibre（PDF 转 EPUB 的备选方案）
brew install --cask calibre

# Claude CLI（核心驱动）
# 确认已安装且能正常调用
claude --version

# jq（JSON 处理，操作术语表）
brew install jq
```

### 2. 创建项目

```bash
mkdir -p ~/translation/book-name/{source,chapters,state/summaries,reviews,output,prompts}
cd ~/translation/book-name
```

### 3. 编写 Prompt 模板

每个阶段有独立的 prompt 文件。翻译 subagent 不需要知道校对怎么做，校对 subagent 不需要知道润色逻辑。职责分离。

#### `prompts/analyze.md`

```markdown
# 科幻小说翻译分析

你是一位资深的英译中科幻小说翻译顾问。你的任务是分析一本英文科幻小说的前几章，产出三份文件供后续翻译使用。

## 输入
- 全书目录（如有）
- 前三章原文

## 产出

### 1. 术语表（glossary.jsonl）
提取所有专有名词，每行一个 JSON 对象：
{"en":"原文","zh":"建议译法","category":"character|location|tech|concept|org","note":"翻译说明","first":"ch01"}

要求：
- 人名：音译为主，参考已有中译本惯例（如有）
- 地名：意译优先，除非是现实地名
- 科技术语：意译为主，保留专业感，避免生造词
- 同一概念如果有多种说法，统一为一种

### 2. 风格指南（style-guide.md）
包含：
- 叙事视角和人称
- 语体定位（书面/口语/混合）
- 时态约定
- 对话翻译原则（贴近口语 vs 保留原文结构）
- 科幻术语处理原则（音译 vs 意译）
- 特殊格式处理（日志条目、通信记录、AI 对话等）
- 需要避免的翻译腔模式

### 3. 世界观摘要（world-summary.md）
200-400 字，涵盖：
- 故事背景（时代、地点、社会结构）
- 核心科技设定
- 主要人物及关系
- 核心冲突
```

#### `prompts/translate.md`

```markdown
# 科幻小说翻译

你是一位优秀的英译中科幻小说译者。你正在翻译一本书的其中一章。

## 输入
- 本章原文
- 术语表（glossary.jsonl）：前面章节已确定的译法，你必须严格遵守
- 风格指南（style-guide.md）：全书的翻译风格约定
- 上一章摘要（如有）：用于衔接上下文

## 翻译要求
1. **术语一致**：术语表中的每个词必须使用表中的译法，不得自行修改
2. **风格统一**：严格遵守风格指南的语体、人称、时态约定
3. **不漏译**：原文每一句都必须有对应译文，不得跳过
4. **自然流畅**：中文要读起来像中文，不是英文的中文皮
5. **上下文衔接**：注意与上一章摘要的情节衔接，不要出现矛盾

## 产出
1. **译文**：完整的中文翻译，保留原文的段落结构
2. **新术语**：本章新出现的专有名词（不在术语表中的），格式同术语表
3. **章节摘要**：200 字以内的情节摘要，用于传递给下一章

## 格式
用以下分隔符分隔三个产出：

--- TRANSLATION ---
（译文）

--- NEW_TERMS ---
（新术语，每行一个 JSON）

--- SUMMARY ---
（章节摘要）
```

#### `prompts/proofread.md`

```markdown
# 科幻小说翻译校对

你是一位严谨的翻译校对。你的任务是对比原文和译文，找出问题。

## 输入
- 本章原文
- 本章译文
- 术语表

## 检查项
1. **漏译**：原文有但译文缺失的句子或段落。逐段比对，这是最重要的检查。
2. **术语不一致**：译文中的术语与术语表不符。
3. **误译**：语义翻反、理解错误、多义词处理不当。
4. **上下文错误**：与世界观设定矛盾的翻译。

## 产出
1. **修订后的译文**：如果发现问题，直接在译文中修正，输出完整的修订版。如果没有问题，原样输出。
2. **校对报告**：列出所有发现的问题和修改。

## 格式
--- REVISED ---
（修订后的完整译文）

--- REPORT ---
（校对报告）
```

#### `prompts/polish.md`

```markdown
# 科幻小说译文润色

你是一位中文文学编辑，专注于科幻翻译的润色。你的任务不是重新翻译，而是让中文更自然。

## 输入
- 本章校对后的译文
- 风格指南

## 润色方向
1. 消除翻译腔："他发现自己正在思考" → "他忍不住想"
2. 拆解欧化长句：一句话超过 40 字的，考虑拆分
3. 对话自然化：让人物说话像真人，不像在念稿
4. 节奏感：动作场景用短句，描写场景可以长
5. 不改术语、不改情节、不改译者确立的核心译法

## 产出
润色后的完整译文。不需要报告，直接输出。
```

### 4. 编写主脚本 `translate.sh`

```bash
#!/bin/bash
set -euo pipefail

# ============================================================
# 科幻小说全自动翻译流水线
# 用法: ./translate.sh source/book.epub
# ============================================================

BOOK="$1"
PROJECT_DIR="$(cd "$(dirname "$0")" && pwd)"
CHAPTERS_DIR="$PROJECT_DIR/chapters"
STATE_DIR="$PROJECT_DIR/state"
REVIEWS_DIR="$PROJECT_DIR/reviews"
OUTPUT_DIR="$PROJECT_DIR/output"
PROMPTS_DIR="$PROJECT_DIR/prompts"

GLOSSARY="$STATE_DIR/glossary.jsonl"
STYLE_GUIDE="$STATE_DIR/style-guide.md"
WORLD_SUMMARY="$STATE_DIR/world-summary.md"
SUMMARIES_DIR="$STATE_DIR/summaries"

# 并行度（校对和润色阶段）
PARALLEL=4

log() { echo "[$(date '+%H:%M:%S')] $1"; }
notify() { claude --print -p "调用 bark 通知：title='[CC] translation' body='$1' id='translation-task' level='$2'" 2>/dev/null || true; }

# ============================================================
# 阶段 0：拆书
# ============================================================
phase_split() {
    log "阶段 0：拆书"

    # EPUB → 单个 Markdown
    local raw="$PROJECT_DIR/source/raw.md"
    pandoc "$BOOK" -t markdown -o "$raw" --wrap=none

    # 用 Claude 按章节标题拆分
    claude --print --dangerously-skip-permissions \
        -p "读取 $raw，按章节标题（# 或 ## 开头的 Chapter/Part 标记）拆分成独立文件。
输出到 $CHAPTERS_DIR/ch01-en.md、ch02-en.md ... 文件名按章节顺序编号。
如果有前言/序言，编为 ch00-en.md。
每个文件保留章节标题。直接操作文件，不需要输出其他内容。"

    local count=$(ls "$CHAPTERS_DIR"/ch*-en.md 2>/dev/null | wc -l | tr -d ' ')
    log "拆书完成，共 $count 章"
}

# ============================================================
# 阶段 1：分析（产出术语表、风格指南、世界观摘要）
# ============================================================
phase_analyze() {
    log "阶段 1：分析"

    # 取前三章（或全部章节如果不足三章）
    local preview_files=""
    for f in "$CHAPTERS_DIR"/ch0{0,1,2,3}-en.md; do
        [ -f "$f" ] && preview_files="$preview_files $f"
    done

    local prompt
    prompt=$(cat "$PROMPTS_DIR/analyze.md")

    claude --print --dangerously-skip-permissions \
        -p "$prompt

以下是需要分析的原文：

$(cat $preview_files)

请将三份产出分别写入以下文件：
- 术语表：$GLOSSARY
- 风格指南：$STYLE_GUIDE
- 世界观摘要：$WORLD_SUMMARY

直接写入文件，不需要额外输出。"

    local term_count=$(wc -l < "$GLOSSARY" | tr -d ' ')
    log "分析完成，初始术语 $term_count 条"
}

# ============================================================
# 阶段 2：翻译（串行，每章一个 subagent）
# ============================================================
phase_translate() {
    log "阶段 2：翻译"

    local prompt
    prompt=$(cat "$PROMPTS_DIR/translate.md")
    local style
    style=$(cat "$STYLE_GUIDE")

    for src in "$CHAPTERS_DIR"/ch*-en.md; do
        local base=$(basename "$src" -en.md)
        local num=${base#ch}
        local dst="$CHAPTERS_DIR/${base}-zh.md"

        # 跳过已翻译的章节（支持断点续跑）
        if [ -f "$dst" ] && [ -f "$SUMMARIES_DIR/${base}-summary.md" ]; then
            log "  $base 已翻译，跳过"
            continue
        fi

        log "  翻译 $base ..."

        # 准备上一章摘要
        local prev_num=$(printf "%02d" $((10#$num - 1)))
        local prev_summary=""
        if [ -f "$SUMMARIES_DIR/ch${prev_num}-summary.md" ]; then
            prev_summary=$(cat "$SUMMARIES_DIR/ch${prev_num}-summary.md")
        fi

        local chapter_text
        chapter_text=$(cat "$src")
        local glossary_text
        glossary_text=$(cat "$GLOSSARY")

        local result
        result=$(claude --print --dangerously-skip-permissions \
            -p "$prompt

## 风格指南
$style

## 术语表
$glossary_text

## 上一章摘要
$prev_summary

## 本章原文
$chapter_text")

        # 解析三段式输出
        echo "$result" | sed -n '/^--- TRANSLATION ---$/,/^--- NEW_TERMS ---$/p' | sed '1d;$d' > "$dst"
        echo "$result" | sed -n '/^--- NEW_TERMS ---$/,/^--- SUMMARY ---$/p' | sed '1d;$d' >> "$GLOSSARY"
        echo "$result" | sed -n '/^--- SUMMARY ---$/,$p' | sed '1d' > "$SUMMARIES_DIR/${base}-summary.md"

        local term_count=$(wc -l < "$GLOSSARY" | tr -d ' ')
        log "  $base 翻译完成（术语累计 $term_count 条）"
    done

    log "翻译阶段完成"
    notify "翻译阶段完成，进入校对" "active"
}

# ============================================================
# 阶段 3：校对（并行）
# ============================================================
phase_proofread() {
    log "阶段 3：校对（${PARALLEL} 路并行）"

    local prompt
    prompt=$(cat "$PROMPTS_DIR/proofread.md")
    local glossary_text
    glossary_text=$(cat "$GLOSSARY")

    local pids=()
    local running=0

    for src in "$CHAPTERS_DIR"/ch*-en.md; do
        local base=$(basename "$src" -en.md)
        local zh="$CHAPTERS_DIR/${base}-zh.md"
        local review="$REVIEWS_DIR/${base}-review.md"

        [ ! -f "$zh" ] && continue

        # 跳过已校对的（支持断点续跑）
        if [ -f "$review" ]; then
            log "  $base 已校对，跳过"
            continue
        fi

        # 控制并行度
        while [ $running -ge $PARALLEL ]; do
            wait -n || true
            running=$((running - 1))
        done

        (
            log "  校对 $base ..."
            local en_text=$(cat "$src")
            local zh_text=$(cat "$zh")

            local result
            result=$(claude --print --dangerously-skip-permissions \
                -p "$prompt

## 术语表
$glossary_text

## 原文
$en_text

## 译文
$zh_text")

            echo "$result" | sed -n '/^--- REVISED ---$/,/^--- REPORT ---$/p' | sed '1d;$d' > "$zh"
            echo "$result" | sed -n '/^--- REPORT ---$/,$p' | sed '1d' > "$review"
            log "  $base 校对完成"
        ) &

        running=$((running + 1))
    done

    wait
    log "校对阶段完成"
}

# ============================================================
# 阶段 4：润色（并行）
# ============================================================
phase_polish() {
    log "阶段 4：润色（${PARALLEL} 路并行）"

    local prompt
    prompt=$(cat "$PROMPTS_DIR/polish.md")
    local style
    style=$(cat "$STYLE_GUIDE")

    local running=0

    for zh in "$CHAPTERS_DIR"/ch*-zh.md; do
        local base=$(basename "$zh" -zh.md)
        local out="$OUTPUT_DIR/${base}-zh.md"

        # 跳过已润色的
        if [ -f "$out" ]; then
            log "  $base 已润色，跳过"
            continue
        fi

        while [ $running -ge $PARALLEL ]; do
            wait -n || true
            running=$((running - 1))
        done

        (
            log "  润色 $base ..."
            local zh_text=$(cat "$zh")

            local result
            result=$(claude --print --dangerously-skip-permissions \
                -p "$prompt

## 风格指南
$style

## 译文
$zh_text")

            echo "$result" > "$out"
            log "  $base 润色完成"
        ) &

        running=$((running + 1))
    done

    wait
    log "润色阶段完成"
}

# ============================================================
# 阶段 5：合并 + 生成 EPUB
# ============================================================
phase_assemble() {
    log "阶段 5：合并输出"

    local merged="$OUTPUT_DIR/book-zh.md"
    > "$merged"

    for f in "$OUTPUT_DIR"/ch*-zh.md; do
        cat "$f" >> "$merged"
        echo -e "\n\n" >> "$merged"
    done

    # 生成 EPUB
    pandoc "$merged" -o "$OUTPUT_DIR/book-zh.epub" \
        --metadata title="译文" \
        --toc \
        --epub-chapter-level=1

    local word_count=$(wc -m < "$merged" | tr -d ' ')
    log "完成！总字数 $word_count，输出：$OUTPUT_DIR/book-zh.epub"
    notify "翻译完成！$word_count 字，EPUB 已生成" "timeSensitive"
}

# ============================================================
# 主流程
# ============================================================
log "开始翻译：$BOOK"
notify "开始翻译 $(basename "$BOOK")" "active"

phase_split
phase_analyze

# 可选：在分析阶段后暂停，让人审核术语表和风格指南
if [ "${AUTO:-1}" = "0" ]; then
    log "分析完成。请审核以下文件后按回车继续："
    log "  - $GLOSSARY"
    log "  - $STYLE_GUIDE"
    log "  - $WORLD_SUMMARY"
    read -r
fi

phase_translate
phase_proofread
phase_polish
phase_assemble
```

### 5. 使用方式

```bash
cd ~/translation/book-name
chmod +x translate.sh

# 全自动模式（一键跑完）
./translate.sh source/book.epub

# 半自动模式（分析后暂停，让你审核术语表再继续）
AUTO=0 ./translate.sh source/book.epub

# 断点续跑（如果中途失败，重新跑会自动跳过已完成的章节）

```

## 运行模式：批处理 + 可选人工卡点

| 模式 | 触发方式 | 覆盖场景 | 你的介入 |
|------|---------|---------|---------|
| **全自动** | `./translate.sh book.epub` | 信任流水线，一键跑完 | 跑完后通读成品 |
| **半自动** | `AUTO=0 ./translate.sh book.epub` | 想在术语定稿前过一眼 | 审核术语表和风格指南后按回车 |
| **断点续跑** | 重新执行同一命令 | 中途失败或想重跑某阶段 | 删掉想重跑的章节文件，再跑一遍 |

这不是交互式的"你说一句它做一步"。这是一个**批处理流水线**——你给它一本书，它还你一本翻译好的书。

---

## 元层：它会越来越好

**术语表跨书复用**：翻完一本书，`glossary.jsonl` 里沉淀了几百条术语。如果你接着翻同一个作者的下一本、同一个系列的续集，直接把术语表拷过去。分析阶段会在现有术语表基础上补充新术语，而不是从零开始。翻第二本书时，"开普勒空间站"已经不需要重新决定了。

**Prompt 迭代**：跑完第一本书，你会发现某些翻译倾向不满意——比如对话太书面、科技描写太啰嗦、某类句式总处理不好。把这些观察写进 `prompts/translate.md` 的翻译要求里，下一本书自动生效。Prompt 模板是 SOP，每一轮使用都是一次验证和打磨。

**校对规则进化**：如果校对阶段老是误报（比如把你有意为之的意译标记为误译），在 `prompts/proofread.md` 里加排除规则。几本书下来，校对 prompt 会精确到几乎零误报。

**风格指南模板化**：翻了几本不同风格的书后，你会积累出几套风格指南模板——硬科幻一套、太空歌剧一套、赛博朋克一套。新书只需要选一个模板作为基础，分析阶段在此基础上微调。

**流水线本身的进化**：你可能会发现四路并行不够（机器性能允许更多）、某些阶段可以合并（校对+润色合为一步）、或者需要加一个新阶段（比如"术语冲突检测"放在翻译和校对之间）。脚本是你的，改几行就能调整流水线结构。这不是一个黑箱产品，是一个你完全掌控的工程系统。
