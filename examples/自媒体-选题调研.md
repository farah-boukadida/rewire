# 场景：自媒体每天花 3-4 小时找选题

> **用户描述**：我做自媒体，每天刷小红书、抖音、微博、知乎找热门内容，找跟我领域（个人成长/效率工具）相关的话题，分析爆款潜力，再写大纲。光找选题和分析就要 3-4 小时，真正写内容只要 1 小时。最烦的是刷平台本身——四个 App 来回切，95% 内容跟我无关，而且刷着刷着就走神了。

---

## 你现在的问题

> 你不是在"找选题"上花了太多时间——你是在用人脑当搜索引擎。四个平台、十几个账号、数万条内容，你在用眼睛做过滤器，用手指做爬虫，用大脑的后台线程跑"这条值不值得看"的判断。3-4 小时里，真正产生价值的判断可能只有 15 分钟，剩下的时间你在做信息搬运工——而且是一个极其容易走神的搬运工。

## 闭环拆解

| 步骤 | 你现在怎么做 | 谁在干 | 问题 |
|------|------------|--------|------|
| **调研** | 刷小红书、抖音、微博、知乎，来回切 4 个 App | 100% 人肉 | 吞吐量瓶颈 + 意愿瓶颈：3-4 小时，95% 是噪音，刷着刷着就走神 |
| **决策** | 直觉判断"有没有共鸣感"，再看点赞收藏验证 | 人（直觉）+ 人肉查数据 | 直觉是真正的判断阀门，查数据是人肉 I/O |
| **规划** | 写大纲 | 人 | 核心方向需要你定，可以辅助 |
| **执行** | 写公众号长文，偶尔改编发小红书 | 人 | 核心产出，1 小时，不是瓶颈 |
| **反馈** | 发布后看阅读/互动数据 | 人肉查后台 | 偶尔看，没有系统性回收 |
| **复盘** | Notion 选题库（经常懒得更新） | 基本缺失 | 选题经验没有固化，每天从零开始 |

调研环节同时是吞吐量瓶颈和意愿瓶颈。更隐蔽的问题是复盘缺失——你没有机制把"哪些选题最终跑出来了"的经验沉淀下来，导致选题判断永远靠直觉，无法系统进化。

## Claude Code 怎么驱动这一切

**机制**：Claude Code 是一个运行在你电脑上的 Agent。它读取项目目录里的指令文件（CLAUDE.md），知道自己的工作是什么、手上有哪些工具。然后它像一个坐在你电脑前的助手一样，通过 shell 命令调用各种工具来完成工作——用 RSSHub 拉取内容源，用 MediaCrawler 抓取平台数据，用脚本处理和评分，用 Notion API 把结果写进你的选题库。每一步都在你的电脑上真实发生。

核心工具链：

- **RSSHub**（[github.com/DIYgod/RSSHub](https://github.com/DIYgod/RSSHub)）— 把社交平台变成 RSS 订阅源。微博 9 条路由（博主动态、热搜、关键词搜索），知乎 24 条路由（热榜、用户动态、专栏、话题），小红书支持用户笔记
- **MediaCrawler**（[github.com/NanmiCoder/MediaCrawler](https://github.com/NanmiCoder/MediaCrawler)，44.9k stars）— 多平台爬虫，覆盖小红书、抖音、微博、知乎，支持关键词搜索和创作者主页抓取，基于 Playwright 保留登录态
- **bb-browser** — 浏览器自动化 CLI，补充抓取和验证
- **Notion MCP Server**（[官方出品](https://github.com/makenotion/notion-mcp-server)）— Claude Code 直接读写你的 Notion 数据库

**故事**：

你早上打开电脑，跟 Claude Code 说一句："帮我看看今天有什么好选题。"

它开始工作。先用 RSSHub 拉取你关注的十几个对标账号最近 24 小时的更新——微博上那几个效率博主发了什么，知乎上"个人成长"话题下有什么高赞新回答，小红书上你关注的几个创作者发了什么笔记。同时，它跑 MediaCrawler，用"个人成长"、"效率工具"、"时间管理"这些关键词在抖音和小红书上搜一轮，把最新的热门内容抓下来。

两分钟后，它手上有了大约 200 条内容。但你不需要看 200 条。它按你定好的规则开始筛选：领域相关性（标题和正文里有没有你的核心关键词）、互动数据（点赞收藏比、评论量）、时效性（24 小时内发布的优先）。筛完，剩 15-20 条候选。

它把每条候选整理成一行摘要：标题、来源平台、核心数据（点赞/收藏/评论）、建议角度。然后它问你："这 15 条里，哪些你想深挖？"

你花三分钟扫一遍。有三条让你眼前一亮——一条小红书笔记讲"用 Notion 管理人生"收藏 2 万，一条知乎回答讲"30 岁后最重要的三个习惯"点赞 5000+，一条抖音视频讲"每天只工作 4 小时"播放量百万。你说："第 3、第 7、第 12 条，帮我展开。"

它读取这三条的完整内容和评论区，分析评论里的高频关键词和情绪倾向，给你生成三份选题简报：原始内容摘要、评论区热议点、可能的切入角度、与你过去选题的差异度。你选了"30 岁后的习惯"这个方向，它帮你拉一份大纲草稿。

最后，它把今天的选题候选和你的最终选择，自动写入你的 Notion 选题库。你的选题库不再是"懒得更新"的摆设，而是一个自动生长的数据库。

**全程，你做了两件事：扫一眼清单，选了三个方向。从打开电脑到拿着大纲开始写，10 分钟。**

## 你的工作会变成什么样

早上坐下来，打开终端，一句话启动。泡咖啡的时间里，Claude Code 已经把四个平台过了一遍。你端着咖啡回来，屏幕上是一份整理好的选题清单——不是 200 条垃圾里找金子，是 15 条已经初筛过的候选，每条带着数据和角度建议。

你花 5 分钟看完，选 2-3 个方向，再花 5 分钟确认角度和大纲。然后你开始写。

一小时后文章写完，发布。

下午你不用刷平台了。你可以去读书、去思考下一篇文章的深度内容、去做真正需要你这颗大脑的事。

那些刷到走神、在四个 App 间来回切换、从海量垃圾信息里肉眼筛金子的日子，结束了。

**每天省下的不只是 3 小时，是 3 小时的注意力损耗和意志力消耗。**

## 怎么搭建

### 1. 部署 RSSHub

```bash
# Docker 一键部署
docker pull diygod/rsshub
docker run -d --name rsshub -p 1200:1200 diygod/rsshub

# 验证：浏览器打开 http://localhost:1200
# 测试微博热搜：http://localhost:1200/weibo/search/hot
```

整理你的对标账号 ID：
- 微博：用户 UID（个人主页 URL 里的数字）
- 知乎：用户 URL ID
- 小红书：用户 ID

### 2. 安装 MediaCrawler

```bash
git clone https://github.com/NanmiCoder/MediaCrawler.git
cd MediaCrawler
uv sync
cp .env.example .env
# 首次运行会弹出浏览器让你扫码登录，登录态会缓存
```

### 3. 接入 Notion MCP Server

1. 去 [notion.so/profile/integrations](https://www.notion.so/profile/integrations) 创建 Internal Integration，拿到 API Token
2. 在 Notion 里把选题库数据库 Share 给这个 Integration
3. 配置 Claude Code 的 MCP：

```json
{
  "mcpServers": {
    "notion": {
      "command": "npx",
      "args": ["-y", "@notionhq/notion-mcp-server"],
      "env": {
        "OPENAPI_MCP_HEADERS": "{\"Authorization\":\"Bearer YOUR_NOTION_TOKEN\",\"Notion-Version\":\"2022-06-28\"}"
      }
    }
  }
}
```

### 4. 配置 Claude Code 项目

创建 `CLAUDE.md`：

```markdown
# 自媒体选题 Agent

## 任务
每天从四个平台收集与"个人成长/效率工具"相关的热门内容，筛选出有爆款潜力的选题候选。

## 信息源

### 对标账号（通过 RSSHub）
- 微博：[你的对标账号 UID 列表]
- 知乎：[你的对标账号 URL ID 列表]
- 小红书：[你的对标账号 ID 列表]

### 关键词监控
- 核心词：个人成长、效率工具、时间管理、自律、复盘、习惯养成

### 抖音（通过 MediaCrawler）
- 路径：/path/to/MediaCrawler
- 关键词：同上

## 筛选规则（v1）
1. 领域相关性：标题或正文含核心关键词 → +3 分
2. 互动数据：点赞 > 1000 或收藏 > 500 → +2 分；点赞 > 5000 → +4 分
3. 时效性：24h 内 → +2 分；48h 内 → +1 分
4. 评论热度：评论 > 100 → +1 分
5. 总分 >= 5 进入候选

## Notion 选题库
- Database ID：[你的数据库 ID]
- 字段：标题、来源平台、原始链接、互动数据、状态、日期
```

### 5. 启动验证

```bash
claude
# 说："帮我跑一遍今天的选题流程"
```

## 元层：它会越来越好

**数据驱动的规则迭代**：每篇文章发布后把实际阅读量、转发量填进 Notion。每周让 Claude Code 对比"候选评分"和"实际表现"，发现规律——比如知乎选题转化率最高，微博热搜阅读高但转发低。然后一起调筛选规则：知乎加权、微博热搜降权。

**对标账号动态管理**：Claude Code 定期扫描同领域涨粉快的新账号，推荐新的关注对象。对标列表不再一成不变。

**筛选规则自动进化**：随着数据积累，从简单阈值进化成更精细的模型：哪些关键词组合最容易出爆款、什么时间段的内容质量最高、评论区什么讨论热度预示有深挖价值。规则写在 CLAUDE.md 里，改一行就生效。
